# 非创伤性SAH面向发表的研究循环

## 1. 论文定位

当前主要研究：

> 利用 MIMIC-IV 3.1 识别危重症非创伤性蛛网膜下腔出血成人患者的早期多模态生理表型及其与结局的关系

该论文应定位为**早期ICU生理表型研究**，而非确定性的输血治疗效果论文。早期贫血和红细胞输注是重要的表型描述指标、探索性效应修饰信号和敏感性分析维度，但主要结论应围绕可解释的早期生理表型及其结局差异展开。

核心研究问题：

> 在非创伤性SAH的成年ICU患者中，常规可获取的0-48小时生理数据能否识别出与死亡率、贫血负荷和输注暴露相关的、临床上可解释的表型？

主要数据来源：

- `mimic-study-498508.non_traumatic_sah_study`
- 源脚本：`10_create_non_traumatic_sah_cohort.sql`
- 分析脚本：`11_bigquery_notebook_non_traumatic_sah_analysis.py`
- 已有策略文档：
  - `docs/non_traumatic_sah_study_transition.md`
  - `docs/early_physiological_phenotype_strategy.md`
  - `docs/next_analysis_steps.md`

## 2. 已确定决策：临床生理特征字典

特征工程必须被视为临床设计产物，而不仅仅是模型输入的列表。

当前已确定的8变量0-48小时低缺失率生理特征面板已在 `10_create_non_traumatic_sah_cohort.sql` 中实现，并由 `11_bigquery_notebook_non_traumatic_sah_analysis.py` 使用。

| 领域 | 主要变量 | 聚合方式 | 恶化方向 | 在论文中的角色 |
| --- | --- | --- | --- | --- |
| 贫血/携氧能力 | `hb_min_48h_all` | ICU入院至48h最低Hb | 降低 | 核心聚类特征；贫血定义为Hb `<10 g/dL`；敏感性分析使用输血前Hb或无RBC队列 |
| 神经损伤 | `gcs_min_48h` | ICU入院至48h最低总GCS | 降低 | 核心聚类特征；主要神经严重程度标志 |
| 神经运动反应 | `gcs_motor_min_48h` | ICU入院至48h最低GCS motor component | 降低 | 核心聚类特征，用于表征神经运动严重程度；需与total-only和GCS grade替代方案比较 |
| 血流动力学灌注 | `map_min_48h` | 最低MAP | 降低 | 核心聚类特征；反映低血压/灌注脆弱性 |
| 血流动力学应激 | `shock_index_max_48h` | 最高HR/SBP | 升高 | 核心聚类特征；反映循环应激 |
| 氧合 | `spo2_min_48h` | 最低SpO2 | 降低 | 核心聚类特征；选择SpO2而非PaO2/FiO2以避免FiO2驱动的数据缺失 |
| 肾脏/器官功能障碍 | `creatinine_max_48h` | 最高肌酐 | 升高 | 核心聚类特征；反映早期肾功能障碍 |
| 血小板/凝血-炎症状态 | `platelet_min_48h` | 最低血小板计数 | 降低 | 核心聚类特征；反映血小板减少/凝血脆弱性 |

候选变量不作为主要聚类输入，除非审查证明其必要性：

| 候选变量 | 当前角色 | 不作为主要变量的原因 |
| --- | --- | --- |
| `lactate_max_48h` | 描述和敏感性子集 | 临床意义明确但常为选择性测量 |
| `pao2_fio2_min_48h` | 氧合敏感性分析 | FiO2和血气缺失可能主导聚类结果 |
| `spo2_fio2_min_48h` | 氧合敏感性分析 | 记录的FiO2覆盖率和单位清洗不稳定 |
| `oxygenation_min_48h` | 氧合敏感性分析 | 复合氧合来源可能混合不同测量机制 |
| `epvs_mean_48h`、`epvs_first_48h`、`epvs_max_48h` | 候选敏感性特征 | 由Hb/Hct衍生；可能与贫血变量共线 |
| `troponin_peak_48h` | 候选机制描述变量 | 检测方法和指征驱动缺失需审查 |
| `sapsiii_24h`、`sofa_24h` | 预测比较/协变量（如有） | 与核心生理变量重叠，且当前表中不可用 |
| `gcs_grade_min_48h` | 描述/敏感性分析比较 | 不纳入主要聚类，因为它直接由total GCS派生 |

特征循环通过标准：

- 每个主要特征具有临床领域、时间窗口、聚合规则、来源表、合理范围过滤和缺失率汇总。
- 所有主要特征的缺失率在可接受范围内。经验规则：`<20%` 可接受，`20%-40%` 需要敏感性分析，`>40%` 应触发替换或降级。
- Hb敏感性通过 `hb_min_48h_pre_transfusion` 和/或无RBC 48h队列明确处理。
- GCS冗余通过 `phenotype_gcs_sensitivity_summary` 明确处理。
- 氧合敏感性在不使FiO2依赖变量成为主要特征的前提下处理。

## 3. 已确定决策：临床可解释表型表

聚类分析的输出必须转化为ICU/神经重症审稿人不依赖算法即可理解的临床表型表。

当前已确定的主要表型方案：

- 主要方案：K=3
- 探索性高分辨率方案：K=4
- 理由：K=3优先考虑稳定性、样本量和可解释性；K=4仅用于检验重度表型内部是否存在一个小型极高危亚组。

必需的主要表型表内容：

| 字段组 | 必需列或来源表 | 目的 |
| --- | --- | --- |
| 规模 | 表型、N、百分比 | 显示是否存在过小的亚组 |
| 结局 | 住院死亡率、ICU死亡率（如可用）、ICU/住院LOS | 显示临床分离度 |
| 核心生理 | 8个主要特征的中位数/IQR | 使表型在临床上可解释 |
| 贫血 | `early_anemia_all`、`early_anemia_pre_transfusion`、Hb分布 | 检验贫血负荷是否因表型而异 |
| RBC暴露 | `any_rbc_transfusion_48h`、`rbc_events_48h`、`rbc_units_48h`（如可靠） | 描述治疗暴露而不夸大因果性 |
| 病因特异性 | `nsah_evidence_level`、动脉瘤诊断/手术标志 | 处理非创伤性SAH异质性 |
| 稳定性 | 轮廓系数、自助法ARI、层次聚类vs K-means ARI | 表明表型不是单一不稳定的算法产物 |

必需的解释规则：

不要仅将聚类标记为P1/P2/P3。每个表型需要基于主导生理特征的临床标签。确切标签需在阅读 `phenotype_cluster_centers_zscore` 和原始汇总后确定，但推荐的标签风格为：

- 低风险/保留生理功能的表型
- 神经损伤主导的表型
- 全身性低灌注或多器官功能障碍的表型

标签必须由数据驱动。例如，只有在某聚类表现出较低MAP、较高休克指数、乳酸升高（如有可用数据）、肾功能障碍或血管活性药物/替代负担增加（如后续添加）的组合时，才应标记为"低灌注"。

表型循环通过标准：

- `phenotype_cluster_centers_zscore` 显示至少两个生理领域之间存在一致的分化。
- `phenotype_outcome_summary` 显示有临床意义的结局差异。
- `phenotype_baseline_characteristics` 未揭示聚类仅为年龄、性别、入院类型或动脉瘤证据的分组。
- `phenotype_gcs_sensitivity_summary` 显示主要结构并非仅为双重GCS编码的产物。
- `phenotype_bootstrap_stability` 的稳定性足够将聚类描述为可重复的。如果ARI较弱，论文必须使用"探索性风险分层表型"而非"已验证的表型"。
- K=4仅在小组具有连贯的临床极端风险特征且不仅仅是离群值或缺失值产物时才被提升。

## 4. 研究循环设计

本项目应以面向论文质量的循环方式运行。每个循环有且仅有一个任务、一个产出物和一个决策。除非变量或模型能解决当前最薄弱的论文问题，否则不添加。

每个循环均设有硬性迭代上限。如果某循环达到上限仍未通过决策规则，将未解决的问题记录为局限性并继续前进——不得无限循环。允许回溯：如果后续循环（如循环6）揭示了需要重新审视早期循环（如循环3）的根本性问题，可返回该循环并重新开始计数。整个项目的回溯次数上限为2次；第三次回溯时，将问题记录为局限性并继续前进。

### 循环1. 研究问题循环

目标：

- 保持论文聚焦于非创伤性SAH早期生理表型。

产出物：

- 一段话的核心假设。
- 暂定标题。
- 目标期刊层次及理由。

决策规则：

- 仅当结论能不包含因果过度推断时继续：早期0-48小时生理数据能够识别与死亡率相关的可解释表型。

失败响应：

- 如果故事主要变成输血异质性治疗效果，将其拆分为后续探索性论文，本文保持表型聚焦。

最大迭代次数：2。若2轮后仍未解决，使用最佳可用框架继续，并将此矛盾记录为局限性。

### 循环2. 队列循环

目标：

- 使队列具有临床可辩解性和可重复性。

产出物：

- `cohort_flowchart_counts`
- 队列流程图
- 纳入/排除标准段落

决策规则：

- 成年非创伤性SAH、ICU住院、首次ICU住院、ICU LOS `>=24h`、主要分析中无大量早期输血、核心特征完整性充分。

失败响应：

- 如果样本损失过大，在修改定义之前先检查是哪个标准导致了损失。不记录临床理由则不得放宽标准。

队列敏感性亚组（在此定义，在循环6执行）：

本循环同时定义循环6将复用的敏感性队列，确保定义一致、避免重复劳动：
- 无RBC 48h队列（前48小时内零红细胞输注的患者）
- ICU LOS `>=48h` 队列（排除早期死亡/早期转出）
- 这些队列在此处一次性定义；循环6将主要分析流程重新应用于这些亚组。

最大迭代次数：3。若3轮后仍未解决，将队列定义固定为最终版本，将剩余的样本损失问题记录为局限性后继续。

### 循环3. 特征循环

目标：

- 锁定8特征主要生理字典和候选敏感性变量。

产出物：

- 第2节中的特征字典表。
- `feature_missingness_summary`
- `oxygenation_source_missingness_summary`
- 候选变量审查：乳酸、ePVS、肌钙蛋白、FiO2依赖的氧合变量。

决策规则：

- 主要特征保持不变，除非缺失率或不可信的分布使某特征无法使用。

失败响应：

- 仅在记录原因后，用缺失率更低且在临床上相邻的变量替换高缺失率的主要变量。

最大迭代次数：3。若3轮后仍未解决，锁定最佳可用特征集，将剩余的缺失率问题记录为局限性。

### 循环4. 表型循环

目标：

- 将K-means输出转化为有临床意义的表型。

产出物：

- `phenotype_cluster_centers_zscore`
- `phenotype_feature_summary_raw`
- `phenotype_outcome_summary`
- 表型标签备忘录

决策规则：

- 如果K=3稳定、可解释且亚组规模适中，保持为主要方案。K=4保持为探索性方案，除非它明确识别出连贯的极端风险亚组。

失败响应：

- 如果聚类不可解释，不要强行贴标签。在尝试更复杂算法之前，重新审视特征集和缺失率。

最大迭代次数：3。若3轮后仍未解决，将最佳可用聚类报告为探索性结果，并相应降低表型结论的措辞力度。

### 循环5. 贫血与输血循环

目标：

- 定义贫血和RBC暴露如何与表型关联，不过度宣称因果性。

产出物：

- `phenotype_anemia_feasibility`
- `phenotype_anemia_stratified_models`
- 无RBC敏感性表型汇总

决策规则：

- 如果表型×贫血的单元格事件数充足，报告调整后的探索性模型。如果单元格稀疏，仅报告描述性事件计数和交互模型。

失败响应：

- 如果RBC暴露罕见或存在强混杂，将输血保持为描述性/敏感性因素，而非主要治疗效应结论。

最大迭代次数：2。若2轮后仍未解决，将输血仅保留为表型描述指标，明确限制因果性语言。

### 循环6. 稳健性循环

目标：

- 使表型结果能够抵御审稿人显而易见的攻击。

本循环复用循环2中定义的敏感性队列（无RBC 48h队列、ICU LOS `>=48h` 队列）。不要在此重新定义——将循环4的主要分析流程应用于这些预定义的亚组即可。

产出物：

- `phenotype_gcs_sensitivity_summary`
- `phenotype_bootstrap_stability`
- `phenotype_sensitivity_cohort_summary`
- `phenotype_epvs_sensitivity_summary`
- 替代聚类方法比较（如可用）

决策规则：

- 主要临床解释应在GCS敏感性、无RBC队列、ICU LOS `>=48h` 队列和K=4高分辨率分析中保持一致。

失败响应：

- 如果某项敏感性分析改变了结论，降低结论的力度，并解释该敏感性分析揭示的问题，而非掩盖它。

最大迭代次数：2。若2轮后仍未解决，如实报告敏感性结果，将其作为已记录的局限性，而不是反复重跑主要分析。

### 循环7. 预测与增量价值循环

目标：

- 表明表型是生理状态的有用汇总，而非仅为装饰性的聚类。

产出物：

- `phenotype_prediction_metrics`
- 校准度 / Brier / AUROC比较

决策规则：

- 仅表型的模型不需要优于所有原始特征。关键问题是：表型+贫血/协变量是否能提供比仅GCS更有临床意义的、可解释的风险分层。

失败响应：

- 如果预测价值较弱，将表型定位为描述性临床特征，避免宣称预测优越性。

最大迭代次数：2。预测模型用于验证而不是生成新假设——两轮足以解决增量价值问题。

### 循环8. 机制循环

目标：

- 将观察到的表型与合理的SAH病理生理机制联系起来。

产出物：

- 机制备忘录和概念图。

机制维度：

- 神经损伤严重程度
- 携氧能力 / 贫血
- 全身灌注储备
- 氧合
- 肾脏和血小板/凝血脆弱性
- 与继发性脑损伤和迟发性脑缺血的可能关联，谨慎表述，因为DCI在MIMIC中可能无法直接观察

决策规则：

- 机制性声明必须直接由表型特征支持，或作为假设提出。

失败响应：

- 如果某机制未被测量，将其写为解释性假设而非发现。

最大迭代次数：2。观察性研究中的机制解释本身带有推测性——两轮足够。

### 循环9. 审稿人循环

目标：

- 在投稿前模拟主要审稿人攻击。

产出物：

- 审稿人质疑-回应表。

必需的审稿人关注点：

- 病例定义和创伤性SAH排除。
- 非创伤性SAH的异质性和动脉瘤证据敏感性。
- 缺失数据和测量机会偏差。
- GCS主导性。
- RBC输血对早期生理表型造成的治疗污染。
- 聚类是否可重复。
- 贫血/输血结论是否被过度解读。

决策规则：

- 每个主要关注点必须对应一个结果表、敏感性分析或明确的局限性说明。

失败响应：

- 如果某关注点无法用数据回应，在投稿前降低论文结论强度。

最大迭代次数：2。两轮审稿人攻击映射后，任何仍未得到回应的问题将成为明确的局限性。

### 循环10. 论文撰写循环

目标：

- 保持写作与分析同步。

产出物：

- 滚动更新的论文大纲。
- 图表框架。

推荐的主要图表：

- 图1：队列流程图。
- 图2：K=3表型标准化中心热图。
- 图3：各表型的死亡率和贫血/输血负荷。
- 补充图：K=4细化及K=3/K=4交叉表。
- 表1：按K=3表型分层的基线特征。
- 表2：原始表型生理和结局。
- 补充表：敏感性分析。

决策规则：

- 摘要中的每一个结果必须能够追溯到某个图表。

失败响应：

- 如果某结果无法清晰展示，从摘要中移除，而非无限制地扩展分析。

最大迭代次数：2。论文同步应快速收敛——如果不能，说明底层的分析循环需要重新审视（回溯）。

## 5. 投稿前停止规则

满足以下所有条件时，研究可进入论文撰写阶段：

- 队列流程可重复且临床可辩解。
- 8特征主要字典已确定并有充分理由。
- K=3表型基于原始和z-score汇总具有可读的临床标签。
- K=4明确描述为探索性高分辨率分析。
- 缺失率、GCS冗余、无RBC队列和ICU LOS `>=48h` 敏感性均已检查。
- 贫血和输血结果以正确的证据级别描述。
- 主要结论避免未经证实的因果性或治疗建议性语言。
- 每个主要审稿人关注点均有一个表格、一项敏感性结果或一条局限性声明与之对应。
