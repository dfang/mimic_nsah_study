# Submission components and decision guide

## Retrieve live requirements

Use authoritative journal and reporting-guideline sites. Record the URL and retrieval date for:

- article type and manuscript structure;
- abstract and word/reference/table/figure limits;
- title-page and blinding rules;
- figure formats and resolution;
- supplement packaging;
- reporting checklist requirements;
- ethics, data/code, preregistration, preprint, and AI policies;
- cover-letter and revision requirements.

Treat third-party summaries as discovery aids, not final authority.

## Select reporting guidance

Base selection on design, not keywords:

- observational EHR cohort: STROBE and, when applicable, RECORD;
- prediction model development/validation: the current TRIPOD-family checklist and journal-required risk-of-bias materials;
- diagnostic accuracy: STARD;
- systematic review/meta-analysis: PRISMA;
- randomized trial: CONSORT;
- quality improvement: SQUIRE;
- economic evaluation: CHEERS;
- causal target-trial emulation: journal requirements plus an explicit target-trial specification.

Complex studies may need more than one checklist. Verify the current version and extensions live.

## Required consistency checks

Cross-check declarations and factual metadata among the manuscript, title page, cover letter, forms, checklist, supplement, registry/protocol, repository release, and submission portal. Any conflict is a blocker until the responsible author resolves it.

## MIMIC-specific disclosure checks

- State the exact MIMIC and linked-module versions.
- Cite each data resource as required.
- Describe credentialed access, training/DUA compliance, and the supplied ethics determination accurately.
- Do not imply that restricted source or derived patient-level data are publicly downloadable.
- Point to public code and explain how an independently authorized user recreates restricted inputs.
